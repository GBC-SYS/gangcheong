## 프로젝트 개요

| 항목       | 내용                                                                 |
| ---------- | -------------------------------------------------------------------- |
| 프로젝트명 | 수련회 미션 트래커                                                   |
| 목적       | 참가자 미션 관리, 신앙 성장 기록, 수련회 운영 개선, 행정 업무 간소화 |
| 타겟       | 강남중앙침례교회 청년들                                              |

## MVP 목표

> 2박 3일 수련회 기간 동안 참가자 개인별 미션 수행 및 신앙 성장 기록

### 핵심 기능

| 우선순위 | 기능             | 설명                                       |
| -------- | ---------------- | ------------------------------------------ |
| P0       | 미션 체크리스트  | 개인별 미션 목록 표시 및 완료 체크         |
| P0       | 간증문 작성      | 수련회 중 느낀 점/은혜 기록 폼             |
| P0       | 설문조사         | 수련회 장단점 피드백 수집                  |
| P1       | 진행 상태 저장   | 로컬스토리지 기반 데이터 유지              |
| P1       | 모바일 최적화    | 청년 타겟, 터치 친화적 UI                  |

### 성공 기준

- [ ] 미션 체크 시 즉시 상태 반영
- [ ] 간증문 작성 후 제출 완료 피드백
- [ ] 설문조사 필수 항목 검증
- [ ] 모바일에서 3초 내 로드
- [ ] 오프라인에서도 체크리스트 사용 가능 (P1)

## 개발 단계

### Phase 1: 환경 설정 및 기본 구조 ✅

- [x] 프로젝트 폴더 구조 생성 (`css/`, `js/`, `assets/`, `data/`)
- [x] index.html 기본 템플릿 작성 (시맨틱 태그)
- [x] CSS 변수 및 리셋 스타일 적용 (`config/design-system.md` 참조)
- [x] 모바일 viewport 설정

### Phase 2: 미션 체크리스트 구현 ✅

- [x] 미션 데이터 구조 정의 (`data/missions.json`)
  ```json
  { "id": 1, "day": 1, "title": "미션명", "description": "설명" }
  ```
- [x] 날짜별 미션 목록 UI (Day 1, Day 2, Day 3 탭/섹션)
- [x] 체크박스 컴포넌트 구현
- [x] 체크 상태 로컬스토리지 저장/불러오기
- [x] 완료율 프로그레스 바 표시

### Phase 3: 간증문 작성 폼 ✅

- [x] 간증문 입력 폼 UI (textarea)
- [x] 작성 가이드 안내 문구
- [x] 임시저장 기능 (로컬스토리지)
- [x] 제출 버튼 및 완료 피드백 (토스트)

### Phase 4: 설문조사 폼 ⏳

- [ ] 설문 문항 데이터 구조 정의 (`data/survey.json`)
- [ ] 설문 폼 UI (라디오, 체크박스, textarea)
- [ ] 필수 항목 검증
- [ ] 제출 완료 처리

### Phase 5: 네비게이션 및 통합 ✅

- [x] 탭 또는 바텀 네비게이션 구현 (미션 / 간증문 / 설문)
- [x] 페이지 간 상태 유지
- [ ] 전체 진행 상황 대시보드 (선택)

### Phase 6: Supabase 연동

- [ ] Supabase 프로젝트 생성
- [ ] 테이블 설계 (missions, testimonies, surveys, users)
- [ ] 간증문 제출 → Supabase 저장
- [ ] 설문조사 제출 → Supabase 저장
- [ ] 미션 체크 상태 동기화 (선택)

### Phase 7: 배포 및 테스트

- [ ] 코드 정리 (console.log 제거, API 키 환경변수 처리)
- [ ] GitHub Pages 배포 (프론트엔드)
- [ ] Supabase Row Level Security 설정
- [ ] 실기기 테스트 (iOS Safari, Android Chrome)
- [ ] 수련회 전 리허설 테스트
